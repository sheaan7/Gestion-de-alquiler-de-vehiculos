Pautas de elaboración

El sistema debe cumplir con los siguientes requisitos:

1. Construcción del Front-End utilizando React:

▸	Componentes React: construir un Front-End modular basado en componentes funcionales. La aplicación debe contener como mínimo 10 componentes funcionales, algunos reutilizables.
▸	JSX y Hooks:
o	Utilizar JSX para definir la estructura de los componentes.
o	Usar los hooks básicos useState y useEffect para manejar el estado y efectos secundarios.
o	Implementar al menos un custom hook que abstraiga lógica compleja o repetitiva.
▸	React Router: implementar un sistema de rutas con React Router, con un mínimo de 4 rutas, como:
o	Página principal: vista general del sistema.
o	Listado de vehículos: mostrar los vehículos disponibles para alquiler.
o	Detalles del vehículo: mostrar información específica de un vehículo.
o	Gestión administrativa: panel para registrar nuevos vehículos o actualizar su estado.

2. Integración con el Back-End:

▸	Consumir las APIs REST desarrolladas en el Back-End mediante fetch o Axios.
▸	Implementar las siguientes funcionalidades:
o	Obtener la lista de vehículos disponibles desde el microservicio "vehículos".
o	Mostrar información detallada de un vehículo específico consultando por su ID.
o	Registrar una solicitud de alquiler interactuando con el microservicio "operaciones".
o	Actualizar el estado de un vehículo tras una operación.
